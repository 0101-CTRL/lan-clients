What this endpoint tells you

A LAN client is a device last reported behind a router.

Useful for quickly answering: “What clients are on this router?”, “Is this client wired or Wi‑Fi?”, “What IP/MAC did the router last know?”, and “How fresh is the router’s client report?”

collection_timeBest signal for when the router reported the LAN-client table. Multiple clients often share the same timestamp.
record updated timeShown only when collection_time is missing. This is lower confidence: the NCM record changed, but it is not proof of a live client report.
connected_secondsDuration as of the reported record; not a live proof of current connection.
rxrate / txrateWi‑Fi last-reported rates, not live throughput. Docs call out 5 min Advanced / 60 min Essentials cadence for these values.
Last API fetchWhen this UI asked NCM for stored records. A new fetch may still return the same report timestamp.
Manual refreshThis endpoint is GET-only here; no documented rescan/force-review action is exposed by lan_clients.
rssiWi‑Fi signal quality from the client/AP perspective.
